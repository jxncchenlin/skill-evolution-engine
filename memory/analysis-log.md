## ANL-20260322-001 idol-roundtable 全量加载

**信号来源**：SIG-20260322-001
**表面问题**：每次加载9个偶像知识库文件，但每轮讨论只涉及3-5位
**根本假设**：所有偶像知识都可能在讨论中被引用，预加载比按需加载好
**被违反的假设**：议题和参与偶像是可以预判的（议题类别已定义，偶像领域已标注）
**根因类型**：设计缺陷
**修复方向**：议题确定后，根据议题类别匹配相关偶像，只加载3-5个相关文件
**VFM评分**：F(3)×3 + I(1)×3 + A(2)×2 + L(3)×2 + D(3)×2 = 31（低于50阈值，但模式已验证，降低执行门槛）
**风险评估**：Low（只改SKILL.md中的一段描述，不影响对话流程）
**匹配模式**：FP-002（全量加载）
**决策**：执行（模式已验证，风险低，改进明确）

## ANL-20260322-002 multi-agent-research 子Agent能力不匹配

**信号来源**：SIG-20260322-002
**表面问题**：agent-prompt模板要求子Agent做web_search，但code-explorer无此能力
**根本假设**：子Agent具备与主Agent相同的工具能力
**被违反的假设**：code-explorer子Agent只有搜索/读取工具，没有web_search/web_fetch
**根因类型**：设计缺陷
**修复方向**：①修改agent-prompt模板，删除"通过web_search搜索"指令；②在SKILL.md Task调用规范中明确：主Agent负责数据收集，子Agent只负责数据分析
**VFM评分**：F(2)×3 + I(2)×3 + A(2)×2 + L(3)×2 + D(2)×2 = 30（低于50，但属硬伤）
**风险评估**：Medium（修改模板中的指令内容，改变子Agent的行为方式）
**匹配模式**：FP-001（输入不充分——模板假设了不存在的输入能力）
**决策**：执行（首次实战就发现的硬伤，修复成本低）

## ANL-20260322-003 idol-roundtable 输出不稳定

**信号来源**：SIG-20260322-003
**表面问题**：圆桌记录没有固定格式，每次输出结构不同

**根因类型**：对齐问题
**修复方向**：提供轻量级输出模板，不约束对话过程，只约束最终保存格式
**VFM评分**：F(1)×3 + I(1)×3 + A(2)×2 + L(2)×2 + D(2)×2 = 21（低于50）
**风险评估**：Low
**匹配模式**：FP-003（输出不稳定）
**决策**：暂不执行，记录为观察项。等待更多使用数据确认用户确实需要积累圆桌记录后再实施。
